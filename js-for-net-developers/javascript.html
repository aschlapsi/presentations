Wenn folgende Werte in einem Bool-Kontext verwendet werden, z.B. als Bedingung in einem if-Statement, ergeben Sie false:

  • 0, -0
  • NaN, ""

Truthy

        console.log(!!true);
        console.log(!!{});

        console.log(!!"Hello, world!");

Zusammenfassung primitive Datentypen

  • Boolean, String, Number
  • Number bildet sowohl int, als auch float ab.
  • JavaScript ist nicht nur dynamisch, sondern auch schwach typisiert.
  • Die Gleichheitsoperatoren vergleichen nach einer automatischen Typumwandlung.
  • Die Identitätsoperatoren berücksichtigen für den Vergleich auch den Typ.
  • Es gibt 7 Werte in JavaScript, die in einem Boolschen Kontext false ergeben.

Best Practices für primitive Datentypen

  • parseInt immer mit Radix aufrufen.
  • Bei Vergleichen nicht den Gleichheitsoperator (==) verwenden, sondern den Identitätsoperator (===).

Objekte in JavaScript...

  • Alles außer primitive Datentypen in JavaScript ist ein Objekt.
  • Sie beinhalten Eigenschaften (Key/Value-Pairs).
  • Sie sind von außen veränderbar.
  • Sie sind vergleichbar mit System.Collections.HashTable.

Iterieren über alle Eigenschaften eines Objekts

this schlägt zurück!

Namenskonvention

        function Animal(name) {
          this.name = name;
        }

        var pet = new Animal("Murli");
        console.log(pet.name);
        console.log(name);

that statt this

        function Animal(name) {
          var that = {};
          that.name = name;
          return that;
        }

        var pet = Animal("Murli");
        console.log(pet.name);
        console.log(name);

Self-invoking Constructor

        function Animal(name) {
          if (!(this instanceof Animal)) {
            return new Animal(name);
          }

          this.name = name;
        }

        var pet = Animal("Murli");
        console.log(pet.name);
        console.log(name);
